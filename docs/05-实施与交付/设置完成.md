## 如何启动游戏

### 方法 1：一键启动（最简单）

双击运行：
```
d:\repo\stardew_valley\start_game.bat
```

这个脚本会：
1. 检查 Ollama 是否运行
2. 验证 qwen3.5:9b 模型
3. 用 Godot 打开项目

### 方法 2：手动启动

```bash
# 1. 确保 Ollama 运行
ollama serve

# 2. 用 Godot 打开项目
D:\program\Godot_v4.6.2-stable_win64.exe --path d:\repo\stardew_valley
```

## 在游戏中

### 测试 AI NPC

1. **按 F5 运行游戏**
2. **走到 NPC 旁边**：
   - Pierre 在 (700, 400) - 商店附近
   - Abigail 在 (400, 300) - 山区附近
   - Lewis 在 (500, 250) - 镇中心
3. **按 E 键对话**
4. **NPC 会用 AI 生成回复！**

### AI 配置界面

1. 点击右上角的 "AI Config" 按钮
2. 可以修改：
   - API Base URL
   - Model Name
   - Temperature（创意程度）
   - Max Tokens（回复长度）
3. 点击 "Test Connection" 测试连接
4. 点击 "Save Configuration" 保存

## 自定义提示

### 想要更快的响应？

编辑 `autoload/ai_agent_manager.gd`：
```gdscript
var api_config = {
    "max_tokens": 128,  # 从 256 改为 128
    "temperature": 0.7  # 从 0.8 改为 0.7
}
```

### 想要不同的个性？

编辑 NPC 场景文件，修改 `ai_personality`：
```gdscript
ai_personality = {
    "traits": ["幽默", "聪明", "神秘"],
    "speech_style": "mysterious",
    "interests": ["魔法", "读书", "星空"]
}
```

### 添加新的 AI NPC？

1. 复制一个现有 NPC 场景
2. 修改 `npc_id`、`npc_name` 和 `ai_personality`
3. 在主场景中实例化

## 故障排除

### AI 没有响应？

```bash
# 1. 检查 Ollama 是否运行
ollama ps

# 2. 确认模型存在
ollama list | findstr qwen3.5

# 3. 重新启动 Ollama
taskkill /F /IM ollama.exe
ollama serve
```

Godot 控制台会显示：
- AI 连接状态
- 模型名称
- 错误信息（如果有）
