# 大场景裁剪 / Chunk 预研（C4）

对应阶段计划 **C4**（仅结论记录，不要求首版实现）。

## 结论摘要

- **触发条件建议**：单场景 **已用 TileMap 单元格数** 或 **同屏可绘实例数** 超过项目自定的阈值（待与真实地图规模联调）时，再启用分区更新。
- **Godot 4.x 方向**：优先利用 **`TileMap` / `TileMapLayer` 的四叉分块渲染** 与 **视锥外剔除**；若仍瓶颈，再考虑按世界坐标切 **Spatial hash / chunk 字典** 存作物与交互物状态，表现层用 **MultiMesh** 或合并 TileMap 区域。
- **存档**：Chunk 化后持久化仍以 **逻辑网格坐标 → 状态** 为主，避免按 Node 路径依赖场景树。
- **与多场景关系**：`WorldRouter` 切换场景时整场景卸载，天然释放大块地图；Chunk 主要服务**单场景极大**（例如未来连体大地图模式）。
