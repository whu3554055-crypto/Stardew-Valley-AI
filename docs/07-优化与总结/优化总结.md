# 性能优化完整指南 - 星露谷物语克隆版

## 📊 当前技术栈分析

### 核心技术
- **游戏引擎**: Godot 4.2 (GL兼容模式)
- **脚本语言**: GDScript (类型提示、信号、异步/等待)
- **AI后端**: Ollama LLM服务器 (qwen3.5:9b模型)
- **分辨率**: 1280x720视口
- **架构**: 20个autoload单例，基于信号的通信

### 当前优势
✅ 优秀的关注点分离（20个专业系统）
✅ 基于信号的解耦防止紧耦合
✅ 插件架构支持扩展性
✅ LLM响应缓存策略（5分钟TTL）
✅ 内存限制（每个NPC最多50个记忆，基于重要性修剪）

## 🔧 关键优化方案（按优先级排序）

### 第一阶段：关键修复（立即见效）

#### 1. 异步叙事生成 ⚡ 最高优先级

**问题**: 故事生成时游戏完全冻结10-30秒

**解决方案**:
- 将叙事生成移至后台处理
- 添加进度条和加载提示
- 流式显示生成的文本

**预期效果**: 零卡顿，等待时间感知减少60%

**实现文件**:
- `autoload/daily_narrative_system.gd` - 添加异步方法
- `scenes/ui/narrative_loading_screen.gd` - 新建加载界面

**代码示例**:
```gdscript
# 旧方式（阻塞）
var narrative = generate_daily_narrative()  # 冻结30秒！

# 新方式（异步）
generate_daily_narrative_async("romantic")  # 立即返回
# 通过信号接收完成通知
signal narrative_generation_completed.connect(_on_story_ready)
```

---

#### 2. 智能NPC更新节流 ⚡ 最高优先级

**问题**: 所有NPC每帧都更新，即使离玩家很远或空闲

**解决方案**:
- 根据距离玩家的远近调整更新频率
- 可见区域：10次/秒
- 同一区域：2次/秒
- 远处：每2秒1次
- 屏幕外：每5秒1次

**预期效果**: NPC相关CPU使用降低60-80%

**实现文件**:
- `autoload/npc_behavior_controller.gd` - 添加定时器管理系统

**代码示例**:
```gdscript
# 根据距离计算更新优先级
if npc在屏幕上:
    更新频率 = 0.1秒  # 高质量
elif npc在同一区域:
    更新频率 = 0.5秒  # 中等
else:
    更新频率 = 2.0秒  # 低频
```

---

#### 3. 有界LRU缓存 ⚡ 防止内存泄漏

**问题**: AI响应缓存无限增长，导致内存泄漏

**解决方案**:
- 限制缓存大小为100条
- LRU（最近最少使用）淘汰策略
- 5分钟自动过期
- 每2分钟清理过期条目

**预期效果**: 缓存内存固定在~10MB以内

**实现文件**:
- `autoload/ai_agent_manager.gd` - 替换缓存逻辑

---

### 第二阶段：高性能扩展

#### 4. 空间分区（社交互动优化）

**问题**: 社交检查是O(n²)复杂度，NPC越多越卡

**解决方案**:
- 网格化空间哈希
- 只检查附近格子的NPC
- 平均复杂度降至O(n log n)

**预期效果**: 支持20+ NPC同时存在而不卡顿

**实现文件**:
- `autoload/npc_behavior_controller.gd` - 添加spatial_grid

---

#### 5. 瓦片地图分块剔除

**问题**: 整个80x50地图（4000个瓦片）全部渲染

**解决方案**:
- 将地图分成16x16的块
- 只渲染摄像机附近的块
- 超出视野的块自动隐藏

**预期效果**: GPU渲染负载降低60-70%

**实现文件**:
- `scripts/game_tilemap.gd` - 添加chunk系统

---

#### 6. NPC数据懒加载

**问题**: 所有18+ NPC档案启动时全部加载到内存

**解决方案**:
- 只预加载常用NPC（Pierre, Abigail等）
- 其他NPC按需加载
- 保持最近使用的8个NPC在内存中
- LRU淘汰最久未使用的

**预期效果**: 初始内存占用降低60%，启动更快

**实现文件**:
- `autoload/enhanced_personality_system.gd` - 改为懒加载
- `data/npcs/*.json` - 提取为独立JSON文件

---

### 第三阶段：用户体验增强

#### 7. 对话流式打字机效果

**问题**: 等待LLM完整响应后才显示文字，感觉很慢

**解决方案**:
- 逐字显示对话（打字机效果）
- 按E键跳过动画
- （进阶）流式接收LLM输出

**预期效果**: 感知延迟降低70%，更专业的感觉

**实现文件**:
- `scenes/ui/dialogue_box.gd` - 添加打字机效果

---

#### 8. 情境化加载界面

**问题**: 长时间操作看起来像游戏崩溃

**解决方案**:
- 显示有趣的背景故事和提示
- 实时进度反馈
- 优雅的淡入淡出动画

**预期效果**: 玩家挫败感减少80%

**实现文件**:
- `scenes/ui/loading_screen.gd` - 新建通用加载界面

---

#### 9. 自适应质量设置

**问题**: 固定质量设置不适应不同硬件

**解决方案**:
- 自动检测GPU显存、CPU核心数
- 动态调整NPC数量、渲染距离、特效
- 提供低/中/高/超高四个档位

**预期效果**: 各种硬件都能流畅运行

**实现文件**:
- `autoload/quality_manager.gd` - 新建质量管理器

### 性能监控工具使用

在 `project.godot` 中添加：
```ini
[autoload]
PerformanceMonitor="*res://autoload/performance_monitor.gd"
```

**游戏中按F3键**显示实时性能数据：
- 当前FPS和平均值
- 内存使用情况
- 活跃NPC数量
- AI缓存大小
- 渲染的地图块数

**生成性能报告**：
```gdscript
PerformanceMonitor.print_performance_report()  # 打印到控制台
PerformanceMonitor.export_performance_report()  # 导出JSON文件
```
