## 当前架构评估

### 核心技术栈现状

```
游戏引擎: Godot 4.2 (GL兼容模式)
脚本语言: GDScript (动态类型)
AI后端: Ollama + qwen3.5:9b (本地部署)
数据存储: JSON文件 (无事务支持)
NPC数量: 6个已集成 + 12个模板
内存占用: ~500MB (持续增长)
```

### 关键问题识别

#### 🔴 P0 - 阻塞性问题（必须解决）

**1. 单线程执行模型**
- **问题**: 所有AI处理在主线程，导致帧率波动
- **影响**: 3个并发请求时帧率降至20 FPS
- **证据**: `advanced_ai_manager.gd` 第181行阻塞等待HTTP响应
- **风险**: NPC数量增加后游戏不可玩

**2. 数据完整性风险**
- **问题**: JSON文件写入无事务保护
- **影响**: 并发写入导致数据损坏
- **场景**: 记忆系统、关系系统同时保存时崩溃
- **风险**: 玩家存档丢失

**3. 内存泄漏**
- **问题**: AI响应缓存无边界增长
- **影响**: 每小时增长~50MB
- **证据**: `response_cache` 字典永不清理
- **风险**: 长时间游戏后内存溢出

#### 🟡 P1 - 重要问题（应该解决）

**4. 查询性能差**
- **问题**: 线性扫描所有记忆（O(n)复杂度）
- **影响**: 50个记忆×30个NPC = 1500次字符串操作/查询
- **场景**: NPC对话时卡顿2-3秒

**5. 无国际化支持**
- **问题**: 所有文本硬编码英文
- **影响**: 无法进入非英语市场
- **工作量**: 后期重构成本是现在的5倍

**6. 插件系统不完善**
- **问题**: 非真正的热插拔，状态丢失
- **影响**: 无法动态更新NPC行为
- **风险**: 扩展性受限

#### 🟢 P2 - 改进性问题（可以延后）

**7. 类型安全性差**
- **问题**: 大量未类型化字典
- **影响**: 运行时错误频发
- **场景**: 拼写错误的键名导致空引用

**8. 对象生命周期管理**
- **问题**: HTTPRequest每次创建新实例
- **影响**: GC压力大，内存碎片化

#### 1.1 引入SQLite数据库

**目标**: 替换JSON文件存储，实现ACID事务

**技术选型对比**:

| 方案 | 优势 | 劣势 | 适用场景 |
|------|------|------|----------|
| **SQLite** | ACID事务、成熟稳定、行业标准 | 需要GDExtension | ✅ 推荐 |
| Redis | 超高性能、缓存友好 | 需要外部服务、持久化复杂 | 不适合单机游戏 |
| PostgreSQL | 功能强大、JSONB支持 | 重量级、需要服务器 | 过度设计 |
| LMDB | 嵌入式、零拷贝 | 学习曲线陡、社区小 | 备选 |

**实施方案**:


**数据迁移脚本**:

```python
# tools/migrate_json_to_sqlite.py
import json
import sqlite3
from datetime import datetime

def migrate_memories():
    # 读取JSON
    with open('user://npc_memories.json', 'r') as f:
        memories_data = json.load(f)
    
    # 连接到SQLite
    conn = sqlite3.connect('user://game_data.db')
    cursor = conn.cursor()
    
    # 迁移数据
    for npc_id, memories in memories_data.items():
        for memory in memories:
            cursor.execute("""
                INSERT OR REPLACE INTO npc_memories 
                (id, npc_id, type, content, importance, emotion, day_created, timestamp)
                VALUES (?, ?, ?, ?, ?, ?, ?, ?)
            """, (
                memory['id'],
                npc_id,
                memory['type'],
                memory['content'],
                memory['importance'],
                memory.get('emotion', 'neutral'),
                memory.get('day', 0),
                datetime.now().timestamp()
            ))
    
    conn.commit()
    conn.close()
    print(f"Migrated {sum(len(m) for m in memories_data.values())} memories")
```

- 查询速度: 1500次操作 → 1次SQL查询（提速99%）
- 数据完整性: 崩溃恢复率 0% → 100%
- 存储空间: 减少40%（二进制vs文本）

#### 1.2 引入向量数据库用于语义搜索

**目标**: 实现智能记忆检索和对话匹配

**技术选型**:

| 方案 | 优势 | 劣势 | 推荐度 |
|------|------|------|--------|
| **ChromaDB (嵌入式)** | Python生态、易用 | 需要Python运行时 | ⭐⭐⭐⭐ |
| **LanceDB** | Rust后端、高性能 | 较新、社区小 | ⭐⭐⭐⭐⭐ |
| **FAISS** | Facebook出品、超快 | 仅向量搜索、无元数据 | ⭐⭐⭐ |
| **SQLite vec0** | 单一数据库 | 实验性功能 | ⭐⭐ |

**推荐方案**: LanceDB（嵌入式、零依赖、高性能）

**实施方案**:

```gdscript
# 通过GDExtension集成LanceDB
# 或使用HTTP API连接独立服务

class VectorMemoryStore:
    var lance_db
    var embedding_model
    
    func _ready():
        # 初始化嵌入模型（使用小型本地模型）
        embedding_model = load_embedding_model("all-MiniLM-L6-v2")
        
        # 初始化LanceDB
        lance_db = lancedb.connect("user://vector_store")
        create_collection()
    
    func create_collection():
        schema = {
            "id": "string",
            "npc_id": "string",
            "content": "string",
            "embedding": "vector(384)",  # MiniLM维度
            "importance": "float",
            "day": "int",
            "metadata": "string"
        }
        lance_db.create_table("memories", schema=schema)
    
    func add_memory(npc_id: String, content: String, importance: float):
        # 生成向量嵌入
        var embedding = embedding_model.encode(content)
        
        # 存储到向量数据库
        lance_db.table("memories").add([{
            "id": generate_uuid(),
            "npc_id": npc_id,
            "content": content,
            "embedding": embedding,
            "importance": importance,
            "day": GameManager.current_day,
            "metadata": JSON.stringify({"emotion": "happy"})
        }])
    
    func search_similar_memories(npc_id: String, query: String, limit: int = 5) -> Array:
        # 将查询转换为向量
        var query_embedding = embedding_model.encode(query)
        
        # 向量相似度搜索
        var results = lance_db.table("memories").search(query_embedding) \
            .where(f"npc_id = '{npc_id}'") \
            .limit(limit) \
            .to_list()
        
        return results
    
    func get_contextual_memories(npc_id: String, context: Dictionary) -> Array:
        """智能获取上下文相关记忆"""
        var query = build_search_query(context)
        return search_similar_memories(npc_id, query, limit=3)
```

#### 1.3 图数据库用于关系网络

**目标**: 高效管理NPC社交网络和关系传播

**技术选型**:

| 方案 | 优势 | 劣势 | 推荐度 |
|------|------|------|--------|
| **Neo4j Embedded** | 成熟、Cypher查询语言 | Java依赖、重量级 | ⭐⭐⭐ |
| **RedisGraph** | 超快、内存中 | 需要Redis服务 | ⭐⭐⭐⭐ |
| **自定义图结构** | 零依赖、完全控制 | 需要自己实现算法 | ⭐⭐⭐⭐⭐ |

**推荐方案**: 自定义图结构（轻量级、适合游戏规模）

**实施方案**:

```gdscript
# autoload/social_graph.gd
class_name SocialGraph extends Node

# 图数据结构
var nodes = {}  # {npc_id: NodeData}
var edges = {}  # {(npc1, npc2): EdgeData}

class NodeData:
    var id: String
    var properties: Dictionary
    func _init(npc_id: String):
        id = npc_id
        properties = {}

class EdgeData:
    var from_node: String
    var to_node: String
    var relationship_type: String  # friend, rival, family, romance
    var strength: float  # 0.0 - 1.0
    var history: Array  # 交互历史
    func _init(from: String, to: String, rel_type: String):
        from_node = from
        to_node = to
        relationship_type = rel_type
        strength = 0.5

func _ready():
    load_graph_from_db()

func add_relationship(npc1: String, npc2: String, rel_type: String, initial_strength: float = 0.5):
    var edge_key = get_edge_key(npc1, npc2)
    if not edges.has(edge_key):
        edges[edge_key] = EdgeData.new(npc1, npc2, rel_type)
    edges[edge_key].strength = initial_strength

func update_relationship_strength(npc1: String, npc2: String, delta: float):
    var edge = get_edge(npc1, npc2)
    if edge:
        edge.strength = clamp(edge.strength + delta, 0.0, 1.0)

func get_social_circle(npc_id: String, max_depth: int = 2) -> Dictionary:
    """获取NPC的社交圈（BFS遍历）"""
    var visited = {}
    var queue = [[npc_id, 0]]  # [node_id, depth]
    
    while queue:
        var current = queue.pop_front()
        var node_id = current[0]
        var depth = current[1]
        
        if visited.has(node_id) or depth > max_depth:
            continue
        
        visited[node_id] = depth
        
        # 找到所有邻居
        for edge_key in edges.keys():
            var edge = edges[edge_key]
            var neighbor = null
            
            if edge.from_node == node_id:
                neighbor = edge.to_node
            elif edge.to_node == node_id:
                neighbor = edge.from_node
            
            if neighbor and not visited.has(neighbor):
                queue.append([neighbor, depth + 1])
    
    return visited

func find_shortest_path(npc1: String, npc2: String) -> Array:
    """查找两个NPC之间的最短路径（BFS）"""
    var queue = [[npc1, [npc1]]]
    var visited = {npc1: true}
    
    while queue:
        var current = queue.pop_front()
        var node = current[0]
        var path = current[1]
        
        if node == npc2:
            return path
        
        for edge_key in edges.keys():
            var edge = edges[edge_key]
            var neighbor = null
            
            if edge.from_node == node:
                neighbor = edge.to_node
            elif edge.to_node == node:
                neighbor = edge.from_node
            
            if neighbor and not visited.has(neighbor):
                visited[neighbor] = true
                queue.append([neighbor, path + [neighbor]])
    
    return []  # 无路径

func propagate_rumor(origin_npc: String, rumor_content: String, max_spread: int = 5):
    """谣言传播模拟"""
    var infected = {}
    var queue = [origin_npc]
    var spread_count = 0
    
    while queue and spread_count < max_spread:
        var current_npc = queue.pop_front()
        
        if infected.has(current_npc):
            continue
        
        # 感染当前NPC
        infected[current_npc] = true
        spread_count += 1
        
        # 告诉朋友
        var neighbors = get_neighbors(current_npc)
        for neighbor in neighbors:
            var edge = get_edge(current_npc, neighbor)
            # 传播概率取决于关系强度
            if randf() < edge.strength * 0.7:
                queue.append(neighbor)
                # 记录谣言传播
                NPCMemorySystem.add_memory(neighbor, {
                    "type": MemoryType.RUMOR,
                    "content": rumor_content,
                    "source": origin_npc
                })
    
    return infected.keys()
```

### 阶段二：性能优化（2-3个月）

#### 2.1 多线程AI处理

**目标**: 将AI推理移至后台线程，消除主线程阻塞

**技术方案对比**:

| 方案 | 复杂度 | 性能提升 | 维护成本 | 推荐度 |
|------|--------|----------|----------|--------|
| **GDExtension (C++)** | 高 | 10x | 中 | ⭐⭐⭐⭐⭐ |
| **Godot Thread API** | 中 | 3x | 低 | ⭐⭐⭐⭐ |
| **外部微服务** | 高 | 5x | 高 | ⭐⭐⭐ |
| **保持现状** | 低 | 1x | 低 | ❌ |

**推荐方案**: Godot Thread API（快速见效）→ GDExtension（长期方案）

**实施方案（Thread API）**:

```gdscript
# autoload/async_ai_processor.gd
extends Node

# 线程池
var worker_threads = []
var task_queue = []
var results = {}
var mutex = Mutex.new()
var semaphore = Semaphore.new()

const MAX_WORKERS = 4

func _ready():
    # 创建工作线程
    for i in range(MAX_WORKERS):
        var thread = Thread.new()
        thread.start(_worker_func.bind(i))
        worker_threads.append(thread)

func _worker_func(worker_id: int):
    while true:
        semaphore.wait()  # 等待任务
        
        mutex.lock()
        if task_queue.is_empty():
            mutex.unlock()
            continue
        
        var task = task_queue.pop_front()
        mutex.unlock()
        
        # 执行任务
        var result = execute_ai_task(task)
        
        # 存储结果
        mutex.lock()
        results[task.id] = result
        mutex.unlock()
        
        # 通知主线程
        call_deferred("_on_task_completed", task.id)

func submit_ai_task(task_type: String, data: Dictionary, callback: Callable) -> String:
    """提交AI任务到线程池"""
    var task_id = generate_uuid()
    var task = {
        "id": task_id,
        "type": task_type,
        "data": data,
        "callback": callback,
        "submitted_at": Time.get_ticks_msec()
    }
    
    mutex.lock()
    task_queue.append(task)
    mutex.unlock()
    
    semaphore.post()  # 唤醒工作线程
    
    return task_id

func execute_ai_task(task: Dictionary) -> Variant:
    match task.type:
        "generate_dialogue":
            return generate_dialogue_sync(task.data)
        "analyze_sentiment":
            return analyze_sentiment_sync(task.data)
        "embed_text":
            return embed_text_sync(task.data)
        _:
            push_error("Unknown task type: " + task.type)
            return null

func _on_task_completed(task_id: String):
    """任务完成回调（主线程）"""
    mutex.lock()
    var result = results.get(task_id)
    mutex.unlock()
    
    if result and result.has("callback"):
        result.callback.call(result.data)
    
    # 清理
    mutex.lock()
    results.erase(task_id)
    mutex.unlock()

# 使用示例
func request_npc_dialogue_async(npc_id: String, context: Dictionary, callback: Callable):
    var task_id = AsyncAIProcessor.submit_ai_task("generate_dialogue", {
        "npc_id": npc_id,
        "context": context
    }, callback)
    
    return task_id
```

**性能提升**:
- 并发能力: 3个请求 → 无限队列
- 主线程FPS: 20 FPS → 60 FPS（AI处理时）
- 响应时间: 30秒阻塞 → 即时返回+异步通知

#### 2.2 对象池系统

**目标**: 减少GC压力，提高内存利用率

**实施方案**:

```gdscript
# utils/object_pool.gd
class_name ObjectPool extends RefCounted

var pool: Array[Node]
var scene: PackedScene
var max_size: int
var created_count: int = 0

func _init(p_scene: PackedScene, p_max_size: int = 20):
    scene = p_scene
    max_size = p_max_size
    pool = []

func acquire() -> Node:
    """从池中获取对象"""
    if pool.size() > 0:
        var obj = pool.pop_back()
        obj.set_process(true)
        return obj
    else:
        # 创建新实例
        created_count += 1
        return scene.instantiate()

func release(obj: Node):
    """释放对象回池"""
    if pool.size() < max_size:
        obj.set_process(false)
        obj.position = Vector2(-9999, -9999)  # 移出视野
        pool.append(obj)
    else:
        # 池已满，真正销毁
        obj.queue_free()

func cleanup():
    """清理所有池对象"""
    for obj in pool:
        obj.queue_free()
    pool.clear()

# 使用示例
# 在main.gd中
var http_request_pool: ObjectPool

func _ready():
    var http_scene = preload("res://scenes/http_request.tscn")
    http_request_pool = ObjectPool.new(http_scene, 10)

func make_api_request(url: String, callback: Callable):
    var http = http_request_pool.acquire() as HTTPRequest
    http.request_completed.connect(func(result, code, headers, body):
        callback.call(body)
        http_request_pool.release(http)
    )
    http.request(url)
```

**内存节省**:
- HTTPRequest对象: 每次创建 → 复用10个实例
- 减少GC频率: 每秒100次分配 → 几乎为零

### 阶段三：架构重构（3-4个月）

#### 3.1 分层架构设计

**目标**: 清晰的职责分离，便于测试和维护

```

│         Presentation Layer              │  ← UI、动画、音效
│  (scenes/, ui/, animations/)            │
├─────────────────────────────────────────┤
│         Application Layer               │  ← 游戏逻辑、状态机
│  (game_logic/, state_machine/)          │
├─────────────────────────────────────────┤
│         Domain Layer                    │  ← 核心业务规则
│  (domain/npc/, domain/economy/)         │
├─────────────────────────────────────────┤
│         Infrastructure Layer            │  ← 数据访问、外部服务
│  (infrastructure/db/, infrastructure/ai)│
└─────────────────────────────────────────┘
```

**示例代码**:

```gdscript
# domain/npc/npc_entity.gd - 纯领域逻辑，无Godot依赖
```

```gdscript
        Provider.CLOUD_QWEN:
```
