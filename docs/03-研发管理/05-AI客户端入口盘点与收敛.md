# AI 客户端入口盘点与收敛

> **目的**：把 Godot 客户端里「谁在调 LLM / 后端」说清楚，支撑 **`04-AI与Agentic开发指导大纲.md`** 里的「划界与收敛」，避免 B 段功能再接出第三条平行管线。  
> **维护**：新增 autoload、或新增直接 `HTTPRequest` 调模型/叙事接口时，更新本文 **§2 一览表** 与 **§4 调用关系**。

---

## 1. 结论摘要（执行层）

| 现象 | 说明 |
|------|------|
| **两套并行「管理器」** | `AIAgentManager` 与 `AdvancedAIAgentManager`（autoload 名 `AdvancedAIManager`）各自维护 **`api_config`、配置文件、以及直连 Ollama 的 `/api/generate`**，范式易碎。 |
| **第三条 HTTP 路径** | `DailyNarrativeSystem` 在「每日叙事」上对 **`AIAgentManager.api_config.backend_url`** 发 **REST**（`/api/v1/narrative/daily`），与「直连 Ollama」并存。 |
| **配置双文件** | `user://ai_agent_config.json` 与 `user://advanced_ai_config.json` 可能漂移（模型、URL 不一致）。 |

**收敛方向（不改变代码也先立规矩）**：

1. **对外 LLM 文本生成**：逐步统一到 **单一入口**（或薄封装层），由该入口决定「直连 Ollama」还是「走后端路由」——见 **§5 迁移表**。  
2. **新功能**（阶段 B）：**禁止**再新增第三套 `HTTPRequest` + 裸 URL；应走统一入口或后端契约。  
3. **`AdvancedAIAgentManager` 的价值**：当前不仅是 HTTP，还包含 **智能体池、日程、社交、异步对话编排**——收敛时建议 **保留能力、合并传输层**，而非简单删类。

## 2. Autoload 与职责一览

| 名称（`project.godot`） | 脚本 | class_name | 主要职责 |
|------------------------|------|------------|----------|
| `AIAgentManager` | `autoload/ai_agent_manager.gd` | `AIAgentManager` | 全局配置（含 `backend_url` + Ollama `base_url`）、健康检查、`generate_dialogue` → `make_llm_request`（**当前实现为直连 `base_url/api/generate`**）、`quick_chat`、缓存、部分 **后端** Agent/缓存 REST |
| `AdvancedAIManager` | `autoload/advanced_ai_manager.gd` | `AdvancedAIAgentManager` | 独立 `api_config`、**直连 Ollama** `make_llm_request`、智能体池、队列、日程/社交、`generate_dialogue_async` |
| `DailyNarrativeSystem` | `autoload/daily_narrative_system.gd` | — | 每日叙事：**优先** HTTP POST 至 **`backend_url/api/v1/narrative/daily`**；本地回退；另有一处 **`AdvancedAIAgentManager.generate_dialogue_async`** 用于脚本生成链路 |
| `AIQuestSystem` | `autoload/ai_quest_system.gd` | — | 任务：若后端可用则走「AI 增强」分支，**当前存在占位/模拟响应**，与真实 `make_llm_request` 未完全接通（见代码注释） |

其他与 AI **相关**但非「生成入口」的 autoload（不重复造 HTTP 客户端时仍需知悉）：`NPCBehaviorController`、`AIEconomySystem`、`AIEventSystem` 等——以各脚本为准。

## 3. 传输路径简图（心智模型）

```
                    ┌─────────────────────────────────────┐
                    │  FastAPI backend (backend_url)       │
                    │  /health  /api/v1/narrative/daily   │
                    │  /api/v1/agent/...  (部分)          │
                    └──────────────┬──────────────────────┘
                                   │
 AIAgentManager.check_backend_status / DailyNarrativeSystem._generate_daily_narrative_from_backend
                                   │
    ┌──────────────────────────────┴──────────────────────────────┐
    │                                                               │
    ▼                                                               ▼
Ollama 直连 (base_url)                                      同上 backend
/api/generate                                               （叙事 JSON）

AIAgentManager.make_llm_request  ──►  base_url/api/generate  （NPC 对话等）
AdvancedAIAgentManager.make_llm_request  ──►  另一份 base_url/api/generate
```

**要点**：同一类「生成文本」请求，存在 **两条 Ollama 直连**（两个 Manager），外加 **叙事走 FastAPI**。收敛的目标不是删掉 Ollama，而是 **调用点与配置来源单一化**。

## 4. 主要调用关系（按脚本）

| 调用方 | 使用的入口 | 备注 |
|--------|------------|------|
| `scripts/npc.gd` | `AIAgentManager.generate_dialogue` / 信号 | 基础 NPC 对话 |
| `scripts/advanced_npc.gd` | `AdvancedAIAgentManager` 注册、日程、`generate_dialogue_async` | 高级 NPC |
| `scripts/npc_behavior_controller.gd` | `AdvancedAIAgentManager` 智能体池、互动 | 行为调度 |
| `scripts/ai_agent_config_ui.gd` | `AIAgentManager` 配置、`quick_chat` | UI 与测试流 |
| `scripts/ai_test.gd` | `AIAgentManager` | 开发测试 |
| `autoload/daily_narrative_system.gd` | `AIAgentManager`（backend URL + 可用性）、`AdvancedAIAgentManager`（脚本生成） | **双栈** |
| `autoload/ai_quest_system.gd` | 意图接 `AIAgentManager`；`generate_ai_enhanced_quest` 内 **仍为占位/模拟** | 待接通或走后端 |

## 5. 迁移 / 收敛表（目标态）

| 当前资产 | 建议目标态 | 优先级 |
|----------|------------|--------|
| 两处 `make_llm_request`（Ollama） | 合并为 **一个** `LLMTransport` / 或只保留 `AIAgentManager` 对外 `request_generate()`，`AdvancedAIAgentManager` 内部调用它 | P1 |
| 两份 `user://*_config.json` | 合并为 **一份**配置，或 Advanced 只读 AIAgentManager 的 `api_config` 子集 | P1 |
| `DailyNarrativeSystem` 自建 HTTP | 保留亦可，但 URL 拼接应来自 **同一配置对象**；长期可改为「后端唯一叙事出口」 | P2 |
| `AIQuestSystem` 模拟 JSON | 接通 **真实校验后的响应** 或 **后端任务生成 API**，与 `04` 大纲 §3.2 对齐 | P2（随 B1） |

**不做的事（避免卡死）**：不要求一次性删除 `AdvancedAIAgentManager`；先 **冻结第三入口**，新代码只挂主入口。

---

## 6. 建议的下一批可验收动作

1. **代码层**：在 `AIAgentManager` 增加 **唯一对外** 的 `request_text_generation(metadata: Dictionary)` 文档注释（可先不拆实现，仅标记调用约定）。  
2. **配置层**：文档或编辑器内说明：**改模型只改 `user://ai_agent_config.json`**，并记录 Advanced 是否与主配置同步的计划。  
3. **B 段**：任务/经济/叙事任一功能，**新增 HTTP** 时在本文件 **§2** 增一行。
